## 🔄 Не транзитивные отношения (Задание 1) — 4 балла ✅

### 💜 Механика «Любимый цвет»

Реализует цикл «камень-ножницы-бумага» через систему любимых цветов с **частичными победами** (модификаторы 0.5×–2.0× вместо бинарных исходов).

#### Правила определения любимого цвета

| Условие                               | Любимый цвет                     |
|---------------------------------------|----------------------------------|
| Один цвет преобладает в руке          | Этот цвет                        |
| Ничья по количеству                   | Цвет последней купленной карты   |
| Purple ≥1 И (Blue == Gold == Red > 0) | **Purple** (специальное правило) |

#### Цикл не транзитивности
```
Purple → Gold → Red → Blue → Purple
```

#### ⚡ Эффекты любимых цветов

| Любимый цвет  | Бонусы                                                                | Уязвимости                                                        |
|---------------|-----------------------------------------------------------------------|-------------------------------------------------------------------|
| 🔵 **Blue**   | Стандартный доход от синих карт всех игроков                          | Красные крадут в **2× больше**; фиолетовые не могут украсть карты |
| 🟡 **Gold**   | Блокирует **50%** краж от красных                                     | Приоритетная цель для фиолетовых (70% шанс)                       |
| 🔴 **Red**    | **Монополия на синие**: доход только активатору; кража у синих **×2** | Штраф **-50%** к золотым картам                                   |
| 🟣 **Purple** | **+50%** к золотому доходу; охотник за золотыми (крадёт 2 карты)      | Не получает доход от чужих синих карт                             |

### 📊 Расчёт вероятностей (Монте-Карло, 10 000 симуляций)

#### Вероятности выбора любимого цвета

| Цвет      | Частота   | WinRate   | Сред. монеты |
|-----------|-----------|-----------|--------------|
| 🔵 Blue   | **42.8%** | 19.4%     | 62.3         |
| 🟡 Gold   | **19.5%** | 31.8%     | 86.0         |
| 🔴 Red    | **37.2%** | **48.2%** | 82.0         |
| 🟣 Purple | **0.5%**  | 0.6%      | 66.2         |

> 💡 **Дизайн-решение**: Низкая вероятность фиолетового (0.5%) — фича, а не баг. Это «легендарная» стратегия с высоким риском и наградой.

#### Матрица ожидаемых значений взаимодействий

**🔴 Кражи (Red-карты, база = 2 корма)**

| Атакующий → Жертва | Базовая кража | Ожидаемая | Примечание                           |
|--------------------|---------------|-----------|--------------------------------------|
| Red → Blue         | 2             | **4.0**   | Blue получает в 2× больше урона      |
| Red → Gold         | 2             | **1.0**   | Gold блокирует 50% (округление вниз) |
| Red → Red          | 2             | 2.0       | Нейтрально                           |
| Red → Purple       | 2             | 2.0       | Нейтрально                           |

**🟣 Кража карт (Purple-карты)**

| Атакующий → Жертва  | Карт к краже | Приоритет  |
|---------------------|--------------|------------|
| Purple → Gold       | **2 карты**  | ✅ Да (70%) |
| Purple → Blue       | **0 карт**   | ❌ Защита   |
| Purple → Red/Purple | 1 карта      | ❌ Нет      |

**💰 Модификаторы дохода**

| Цвет карты | Любимый цвет | Модификатор      | Пример (база=1)                |
|------------|--------------|------------------|--------------------------------|
| Gold       | Purple       | ×1.5             | 1 → **1.5**                    |
| Gold       | Red          | ×0.5             | 1 → **0** (округление вниз)    |
| Blue       | Red          | Только активатор | Другие не получают доход       |
| Blue       | Purple       | 0 от чужих       | Получает только от своих синих |

### 🔢 Формула расчёта эффективности стратегии

```
Эффективность(фокус F) = 
    Σ[ P(цвет_раунда) × P(карта_в_руке) × Модификатор_взаимодействия ]

Пример для Red против Blue:
  = P(Red) × P(иметь_красную) × 2.0(бонус_кражи)
  = 0.227 × 0.35 × 2.0 ≈ 0.159 ожидаемого преимущества за раунд
```

## 💰 Мета-экономический пласт (Задание 2) — 4 балла ✅

### 📐 Схема мета-экономики

```
┌─────────────────────────────────────────┐
│         МЕЖДУ ИГРАМИ (Meta-Layer)       │
├─────────────────────────────────────────┤
│                                         │
│  🎮 Завершение партии                   │
│           ↓                             │
│  🏆 Расчёт мест + бонусы за сеты цветов │
│           ↓                             │
│  💎 Начисление репутации:               │
│     • 1 место: +50 репутации            │
│     • 2 место: +30 репутации            │
│     • 3 место: +15 репутации            │
│     • 4+ место: +5 репутации            │
│     • +5 за каждый сет из 3+ карт цвета │
│           ↓                             │
│  📁 Сохранение в Data/meta_profiles.json│
│                                         │
└─────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────┐
│         СЛЕДУЮЩАЯ ПАРТИЯ                │
├─────────────────────────────────────────┤
│                                         │
│  🎁 Применение бонусов при старте:      │
│     • Каждые 100 репутации = +1 монета  │
│     • Максимум: +2 монеты к старту      │
│                                         │
│  📊 Прогрессия:                         │
│     • Игрок чувствует рост между сессиями│
│     • Нет софтлока: бонусы умеренные    │
│                                         │
└─────────────────────────────────────────┘
```

### 🔢 Расчёты баланса мета-экономики

**Формула стартового бонуса:**
```
StartCoinBonus = min(2, floor(Reputation / 100))
```

**Пример расчёта:**
```
Игрок "Константин":
• Репутация: 247
• Бонус: min(2, floor(247/100)) = +2 монеты
• Стартовый капитал: 5–10 (рандом) + 2 = 7–12 монет

Эффект на игру:
• +2 монеты = возможность купить карту за 2 корма раньше
• Не ломает баланс: разница в 2 монеты << целевых 100
```

**Ожидаемый импакт:**
| Репутация | Бонус | Влияние на геймплей |
|-----------|-------|-------------------|
| 0–99 | +0 | Базовый опыт |
| 100–199 | +1 | Ранняя покупка карты за 1 корм |
| 200+ | +2 | Ранняя покупка карты за 2 корма |

> ⚖️ **Баланс-принцип**: Бонусы дают тактическое преимущество, но не гарантируют победу. Игрок без репутации может выиграть за счёт стратегии.

### 💾 Реализация

- **Файл**: `Services/MetaService.cs`
- **Хранение**: `Data/meta_profiles.json` (JSON, человекочитаемый)
- **Интеграция**: Бонусы применяются в `GameSessionService.InitializePlayers()`

### 📈 Процесс балансировки

#### Этап 1: Базовый анализ (Задание 2)
- Метод Монте-Карло: 1 000 симуляций
- Выявлены карты с окупаемостью <1 хода («Стипендия»)
- **Действие**: Снижен вес карты с 50 → 40

#### Этап 2: Внедрение не транзитивности
- Добавлены модификаторы любимых цветов
- **Риск**: Красные карты стали слишком сильными против синих
- **Действие**: Добавлена защита для золотых (блокировка 50% краж)

#### Этап 3: Финальная полировка (текущая)
- Увеличено количество симуляций до **10 000** для точности
- Корректировка весов карт на основе эмпирических данных
- Балансировка частот активации цветов

## 🚀 Запуск проекта

### 📋 Требования
- .NET 8 SDK
- Файл `Cards.xlsx` в корне проекта
- Изображения карт в `wwwroot/images/cards/`

### ▶️ Быстрый старт
```bash
# 1. Клонирование и восстановление
git clone https://github.com/RelaxerR/TinyCityCardGame_online.git
cd TinyCityCardGame_online
dotnet restore

# 2. Запуск сервера
dotnet run

# 3. Игра
#    Хост: открыть http://localhost:5090 → создать комнату
#    Гость: ввести имя + код комнаты → присоединиться
```

### ⚙️ Настройка баланса (`appsettings.json`)
```json
{
  "GameSettings": {
    "WinTarget": 100,
    "DailyIncome": 1,
    "StartCoinsMin": 5,
    "StartCoinsMax": 10,
    "ColorChanceBlue": 50,
    "ColorChanceGold": 30,
    "ColorChanceRed": 25,
    "ColorChancePurple": 5,
    "MarketSizeFormula": "{players_count} + 1",
    "MaxRounds": 100
  }
}
```
